# Phase 4A - AI Screening Foundation

Phase 4A adds the data model and UI for AI screening without requiring a live OpenAI key yet.

## Added Tables

- `ai_screening_templates`
  - Stores reusable screening prompts, questions, and scoring rubric by role family.

- `ai_screening_tasks`
  - Stores candidate-specific AI screening status, context, structured scores, summary, recommendation, and risk flags.

## Frontend Updates

- Applicant detail pages now include an `AI screening assessment` panel.
- The panel shows:
  - task status
  - role fit score
  - professionalism score
  - communication score
  - availability score
  - AI summary
  - recommendation
  - risk flags

## Automation Updates

- New application submissions now create an `ai_screening_tasks` record.
- New application submissions now queue an `evaluate_ai_assessment` automation job.
- The local fallback processor and Supabase Edge Function can process `evaluate_ai_assessment`.
- Processing this job writes placeholder structured screening output and moves the applicant to `Assessment Completed`.

## Still Placeholder

- No OpenAI API call yet.
- No real prompt execution yet.
- Scores and summary are generated by deterministic placeholder logic.
